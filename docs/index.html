企画書

【タイトル】
通勤アラーム
【はじめに】
私自身、現在の勤務先が横浜であり、通勤時間が片道2時間とかなり長時間です。 そのため、働き方改革により、残業時間が短くなったとはいえ、通勤時間を考慮すると、帰宅が遅くなってしまいます。 朝も、出勤時間に合わせるため、電車遅延などの不測の事態も考慮すると、家を出る時間も早くせざる追えず、どうしても睡眠時間が少なくなってしまいます。 そこで、長時間の通勤時間が貴重な睡眠時間としたいのですが、寝入ってしまい、寝過ごしてしまうと、次の駅までの駅間が長く、出社時間に間に合いません。 そこで、駅を選択し、到着したらアラームがなるようアプリが存在したら、乗り過ごしを考えることなく、貴重な睡眠が取れると考えました。
【現在のアプリ】
現在も、GPSを使用した類似のアプリは存在するが、GPSである限り、天候が悪かったり、地下鉄などでは精度が悪く、使用することができません。 時間でのアラームでは、遅延した際にかなり前に起こされてしまったり、その後どのくらいで到着するかがわからないので、結局意味を成しません。
【目的】
天候や地下に左右されず、駅に到着したことを判断できる仕組みを検討し、構築すること。 実施案 Wifiの電波や、基地局情報から、現在の位置特定（DGPS等）を行い、駅にいることを判断することを検証、検討する。 もしくは、壮大な計画となるが、電車の中に、駅情報を配信する仕組みを設け、その情報を取得する。
【課題】
・Wifiや基地局も情報を元に、駅の特定が可能かどうか
・電車へは現在位置を特定する情報は送られているはずなので、その情報を一般に使用できないかどうかを検討する。
